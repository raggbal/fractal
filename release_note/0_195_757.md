# Release 0.195.757 — md / table / sidepanel batch (v16)

## ハイライト

1. **Markdown テーブルの列幅 D&D resize がどの行のセルからでも可能に** — ヘッダー以外の任意 cell の右端でドラッグして列幅変更。列幅は `<!-- fractal-col-widths: ... -->` コメントで永続化され、ファイルを開き直しても保持される。
2. **画像と drawio.svg の最大幅を `fractal.imageMaxWidth` 設定で制御** — デフォルト 600px。エディタ幅いっぱいに広がる挙動を任意のサイズに制限可能。
3. **Side panel の MD ナビゲーション履歴 (戻る / 進む)** — Side panel 内で MD link をクリックして遷移した履歴を ←/→ ボタン or Opt+Left/Right で行き来できる。
4. **cmd+/ Add Page でリンク名を入力できるモーダル + 全 context で simple flow** — 旧 action panel 多段確認を廃止。任意の名前を入力 → page MD と link が即座に挿入される。空 heading (`# `) も可視化されるため、新規 page の H1 を即タイトル入力できる。
5. **drawio.svg 外部編集の取りこぼし対策** — drawio Desktop の atomic-rename 保存が VSCode FileSystemWatcher で取りこぼされる問題を `fs.watchFile` polling fallback で確実に検知。複数 drawio.svg を同じ MD に置いた時の 2 つ目以降の更新失敗が解消。

## 詳細な変更内容

### 新機能

#### `fractal.imageMaxWidth` 画像最大幅設定 (MD-52)
- 数値型、default `600` (px)、min `100`
- 標準のエディタ / Side panel / Outliner page の Side panel すべてに適用
- `<img>` の inline `style="max-width:100%"` を CSS rule (`!important`) で上書き
- toolbar/lucide/command-palette icon は除外
- ダブルクリックでフルスクリーン表示は既存機能のまま動作

#### cmd+/ Add Page リンク名入力モーダル (MD-49)
- VSCode webview の sandbox で `window.prompt` がブロックされるため、カスタム overlay モーダルで実装
- default `untitled`、Enter で確定、Escape / Cancel で marker 削除
- linkName が `untitled` 以外なら新規 MD の H1 を自動で同期 (`host.updatePageH1`)
- `useSimpleAddPage = true` (常時) — 旧 action panel 多段確認 (auto/at-path) は廃止し、すべての context で同じ simple flow に統一

#### 右クリック → リンク名変更 (MD-50)
- `<a href>` (md link / file link / URL link) 上で右クリック → 「リンク名変更」メニュー項目
- 同じ custom modal でリンク名 (textContent) を編集 — href は変更されない
- 7 言語対応 (`contextRenameLink` / `promptRenameLink`)
- context menu の separator を theme-aware に (light theme でも目立たない灰色)

#### Side panel 戻る / 進むナビゲーション (SP-01)
- ヘッダー左 (Open Outline ボタンの右、ファイル名の左) に ←/→ ボタン追加
- ショートカット: **Opt+Left / Opt+Right** (Mac 標準)
- 履歴がない時は半透明 (opacity 0.5)、navigatable 時は不透明
- Side panel 内 MD link クリックで履歴に push、Outliner / 外部からの fresh open で履歴クリア

#### Side panel outline 常時表示 (SP-02)
- 見出しが 1 つも無い MD でも outline は default ON
- 空 placeholder「見出しがありません」(7 言語、`outlineEmpty`)
- ユーザーが明示的に閉じた時のみ非表示

#### Side panel cmd+/ Add Page (SP-03)
- Side panel 内の MD でも cmd+/ → Add Page で新規 page を作成可能
- pageDir: outliner page を開いている時は outliner pageDir、そうでなければ `<sidePanelDir>/pages`
- 新規 MD は `<timestamp>.md` で自動命名、初期 content `# ` (空 heading 可視化と組み合わせて即タイトル入力可)

#### 空 heading の可視化 (MD-51)
- `# ` (trailing space のみ、テキスト無し) を `<h1><br></h1>` で render
- これにより cmd+/ Add Page で作成した直後の page を開くと h1 が見え、即タイトル入力できる

#### Markdown テーブル列幅 D&D resize (TBL-01〜03)
- **任意の cell の右端**でドラッグして列幅変更 (ヘッダー行に限らない)
- 視覚: hover/D&D 中は列全体が同じ青で連続表示 (行間ギャップ無し)
- ヒット領域 4px、mouse 追従 ±3px、min-width 80px clamp
- 新規行 (Enter / Add Row) でも自動的に動作
- 列幅は `<!-- fractal-col-widths: w1,w2,w3 -->` コメントで markdown に保存され永続化
- 他 markdown viewer (GitHub 等) はコメントを無視するので互換性維持

### 改善

#### drawio Desktop 外部編集の取りこぼし対策 (MD-53)
- 旧 `vscode.workspace.createFileSystemWatcher` のみではプラットフォーム依存で drawio Desktop の atomic-rename (write tmp → rename) を取りこぼすケースがあった
- `createDrawioFileWatcher` factory を新設、`vscode.workspace.createFileSystemWatcher(RelativePattern)` + `fs.watchFile(path, {interval:1000})` polling fallback を統合
- `editorProvider.ts` (standalone) / `notesEditorProvider.ts` (Notes) 双方で使用 — Notes mode に欠けていた polling fallback を追加
- webview 側 matcher: 絶対 path full match 優先 + basename fallback の二段判定。同 mtime 重複通知時は強制 reload (`removeAttribute → setAttribute`)

#### Side panel cmd+/ → drawio 挿入の routing fix (MD-54)
- `SidePanelHostBridge.requestCreateDrawio()` が `_onImageRequest()` を呼んでいなかったため `sidePanelImagePending=true` がセットされず、`insertImageHtml` response が main editor に流れて side panel に挿入されない問題を修正

#### cmd+/ Add Page / drawio 挿入の selection robustness (MD-54)
- selection が editor 外を指している時の挿入失敗を防ぐ
- `editor.contains(startContainer)` 検査 + 外なら editor 末尾に新規 `<p>` で marker を pin
- `insertImageHtml` / `insertFileLink` / `insertLinkHtml` の fallback でも同等の防御

#### Table 一番右の列を resize しても他列が縮まない (TBL-02)
- 旧実装は `th.offsetWidth` (browser layout 後の値) ベースで再計算 → `table-layout: fixed` の自動縮小と相互作用して連鎖縮小
- 新実装: 全列の `style.width` を array で管理 → 合計を**先に** `table.style.width` に反映してから cell の `style.width` を再設定

#### Table 範囲選択 cmd+c → cmd+v で空 cell 行が壊れない (TBL-04)
- `normalizeMultiLineTableCells` の de-flatten 正規表現が正当な空 cell 行 `| <br> | <br> |` まで誤って row 分割していた
- 修正: 同一行内に `| <br> | --- |` パターン (Notion-flattened の signature) を含む時のみ de-flatten 適用 (line-by-line)

#### Table 空 cell の `<br>` を markdown 出力時に除去 (TBL-05)
- 新規行の空 cell `<td><br></td>` (contenteditable 表示用に必要) を markdown serialize 時には空セル `|  |` として出力
- 中間 `<br>` (`text<br>text` で line break) は保持

#### Table cursor / Shift+Enter の根本修正 (TBL-06, TBL-07)
- 旧実装の `<div class="table-col-resize-handle" contenteditable="false">` を cell 内 DOM に置く方式は contenteditable のカーソル navigation を trap し、ArrowRight で右端居着き / 空 cell Shift+Enter で `<br>` 過剰挿入の症状があった
- **DOM handle を完全廃止 → CSS `::after` pseudo-element + マウス位置判定**方式に置換
- cells に余計な要素なし → cursor navigation は通常通り
- 副次効果: side panel re-init 時の `Object.defineProperty(window, 'activeTableCell', ...)` を `configurable: true` に変更 (`TypeError: Cannot redefine property` 防止)

### 修正

- Markdown table の copy-paste で空 cell 行が orphan `|` line に壊れる問題 (TBL-04 で根本対策)
- Side panel から MD link を開いた直後に back ボタンを押しても何も起きない問題 (SP-01)
- 標準 MD で cmd+/ → "Add Page" 多段確認の途中でユーザーが操作を見失う問題 (MD-49: 単一 modal に統合)
- 複数 drawio.svg を含む MD で 2 つ目以降の外部編集が反映されないケース (MD-53)

## アップグレード時の注意

- `fractal.imageMaxWidth` (default 600px) の影響で、これまでエディタ幅いっぱいに表示されていた画像が **600px に縮む** ように見える可能性。元の挙動が好みなら `fractal.imageMaxWidth` を大きな値 (例: 2000) に設定。
- Markdown table の resize 後の列幅が `<!-- fractal-col-widths: ... -->` コメントとして MD に書き込まれる。気になる場合はコメント行を手動で削除すれば自動 layout に戻る。
- cmd+/ → Add Page の挙動が変わり、リンク名入力 modal が出るようになった (Cancel で何もしない、OK で挿入)。
