# Release v0.195.718

**リリース日:** 2026-04-26
**バージョン種別:** patch (bugfix + observability + outliner UX)
**対応要件:** v14 — Sprint `20260424-135027-debug-banner-outliner-actions`

## 概要

v0.195.717 で導入した debug banner を console-only 化しつつ、ユーザー実機で再現確認した「single MD + side panel 同時編集 + 別アプリ切替で disk からデータ消失」現象に対する **view rollback hotfix (Fix A)** を投入。あわせて outliner ノードに対する 2 つの操作拡張（右クリック "Copy File Path" / Cmd+Enter で外部アプリ起動）を追加。

## 変更内容

### Added

#### Outliner: "Copy File Path" コンテキストメニュー (FR-OL-COPYPATH-1)
- file 添付ノード (`node.filePath` あり) を右クリック → **"Copy File Path"** で添付の絶対 path を OS clipboard へ copy

  - `outliner-host-bridge.js` / `notes-host-bridge.js` (送信側)
  - `outlinerProvider.ts` / `notes-message-handler.ts` (受信 handler)
- md page ノードは既存 "Copy Page Path" が同等機能を提供 (重複追加なし)
- 添付なしノードには表示しない

#### Outliner: file 添付ノードでの Cmd+Enter (FR-OL-CMDENTER-1)
- outliner 上で file 添付ノードにフォーカス → `Cmd+Enter` (Mac) / `Ctrl+Enter` (Win/Linux) で OS デフォルトアプリ起動
- 既存 `host.openAttachedFile(nodeId)` を再利用
- md page ノード (`isPage`) は既存 `openPage` (side panel 表示) を完全維持
- 添付なしノードは preventDefault のみで挙動変更なし
- `isPage` と `filePath` は data-model §4.2 で相互排他

#### Editor blur 観測 (FR-DBG-1, 診断用)
- `editor.blur` / `sourceEditor.blur` / `_handleVisibilityChange` (hidden) で `hasUserEdited && queuedExternalContent !== null` 同時成立時、`console.warn '[Fractal:blur-with-queue]', { instance, domLen, queueLen, delta }` を出力
- v0.195.717 で導入した UI banner は本リリースで削除（DevTools での観測のみ残置）
- 通常運用 (queue が空 or 未編集) では一切実行されない (early return) — オーバーヘッド 0

### Fixed

#### View rollback hotfix (FR-DBG-FIX-A)
- **症状**: single MD + side panel 同時編集中に別アプリ切替 → 戻ってきた時に DOM が古い queue 内容に巻き戻り、user が再編集すると disk が「巻き戻った文字列＋新しい打ち込み」で上書きされ、user が打っていたはずの文字が消える (5-6 回再現)
- **修正**: `editor.blur` / `sourceEditor.blur` / `_handleVisibilityChange` で `hasUserEdited` triggered flush 時、`queuedExternalContent = null` で stale queue を破棄し、`applyQueuedExternalChange()` を `if (!didFlushUserEdit)` で条件付き化
- **正当性**: user が直前まで編集していた typing が「真」、cross-edit 由来の queue は古い content。drop した queue 内容は cross-edit の round-trip で次の event 時に再到来するため **NT-14 (複数パネル間同期)** は維持される
- **影響範囲**: webview 内 blur ハンドラのみ。host 側の `editorProvider.onDidChangeTextDocument` / `sidePanelManager.onDidChangeTextDocument` listener は不変

## 既知制約 (Sprint v15+ で個別調査)

Fix A の `if (hasUserEdited && queue !== null)` ガードでは catch しない以下の edge case では、別経路で view 不整合の可能性が残る:

- **IME 変換中の app switch**: composition state が `hasUserEdited` を立てるタイミング不確定で、ガード条件にヒットしない可能性
- **debounce 1000ms 内の超高速 typing 直後の app switch**: `host.syncContent` 未発火のまま blur した場合、disk への flush タイミングがずれて未同期文字を失う可能性

**回避策（暫定）**:
- 別アプリへ切り替える前に ~1 秒待つ
- もしくは Cmd+S を明示的に押してから切り替える
- `[Fractal:blur-with-queue]` console log は edge case でも出力される (= 検出は可能、修正は v15+ で)
