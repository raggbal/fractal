# Release v0.195.706

**リリース日:** 2026-04-14
**バージョン種別:** patch
**対応要件:** なし（チャットベースの小修正）

## 概要

MD エディタからリスト（`- item` 形式）をコピーして outliner に貼り付けた際、各ノードの先頭に `- ` というバレット記号が残っていた問題を修正。

## 変更内容

### バグ修正
- **Outliner: MD リスト貼り付けでバレットが残る** — MD エディタで `- foo\n- bar\n- baz` のようなリストをコピーして outliner にペーストすると、各ノードのテキストが `- foo`, `- bar`, `- baz` になっていた。outliner はノード自体がバレット表示をするため、MD のバレット記号は冗長。修正:
  - `pasteNodesFromText` (outliner 複数行ペースト処理) で、インデント除去後の content から先頭の MD リストマーカー (`- `, `* `, `+ `, `1. ` 等) を strip
  - 内部 outliner-to-outliner コピー (`clipboardNodes` あり) は従来通りノードテキストをそのまま保持
  - 単一行ペースト (カーソル位置挿入) は影響なし — 対象は複数行ペーストのみ

## 実装詳細

### 変更ファイル一覧

| ファイル | 変更種別 | 内容 |
|---|---|---|
| `package.json` | 修正 | version 0.195.705 → 0.195.706 |
| `src/webview/outliner.js` | 修正 | `pasteNodesFromText` で `!clipboardNodes` の場合、`content.replace(/^(?:[-*+]\|\d+\.)[ \t]+/, '')` でバレット除去 |
| `CHANGELOG.md` | 修正 | v0.195.706 エントリ追加 |

### 主要な実装判断
- **内部コピーと外部ペーストを区別**: `clipboardNodes` の有無で判別。既存の URL 自動変換ロジック (`convertUrlsToMarkdownLinks`) と同じ分岐条件を流用。内部コピーの node.text は意図的に保持されているべきなので、外部ペースト時のみ正規化する。
- **正規表現をタイト化**: `[ \t]+` でスペースまたはタブの必須一致。バレット直後に空白が来ない場合 (例: `-dashed-word`) はハイフンを保持。
