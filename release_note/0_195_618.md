# 0.195.618

## 概要
検索結果に「未リンクページ」が大量に出る + そのページがクリックしてもジャンプしない問題の根本修正。

## 課題
AWS note 等で「未リンクページ」が27件など大量に表示され、それらをクリックしても何も起きない（サイドパネルが開かない、ノードジャンプも発火しない）。

## 原因
[notes-file-manager.ts:818-851](src/shared/notes-file-manager.ts#L818-L851) の旧 `searchFilesStreaming` は次のフローだった:

```ts
for (outline of outFiles) {
  pDir = outline.pageDir or default(./pages)
  pageIdToNodeText = build map from outline.nodes
  pageMds = readdirSync(pDir)               // ← ディレクトリ内 .md 全列挙
  for (pm of pageMds) {
    nodeText = pageIdToNodeText.get(pm)     // 引けなければ「未リンクページ」
    onResult(...)
  }
}
```

問題:
1. **ディレクトリ全列挙**: outline 所有でない .md (true orphan、または ノードの text が空で逆引きが空文字になるケース) も結果に含まれる
2. **ジャンプ不能**: 結果に出ても、`notesJumpToMdPage` の逆引き ([notes-message-handler.ts:634-641](src/shared/notes-message-handler.ts#L634-L641)) で `pageNodeId` が null になり、`if (pageNodeId)` が false → サイドパネルも開かず `jumpToNodeId: null` → outliner も動かない。**未リンクという状態自体が完全に詰み**

3. **重複ヒット**: 旧仕様で pageDir が共有されていたケースでは、同じ .md が outline 数ぶん重複ヒットしていた

## 対策

### 設計
ディレクトリを readdir する方式を完全廃止し、**outline の `nodes` を走査して `pageId` を持つノードだけを検索対象にする**。これにより:
- 結果に出るすべてのページは必ず有効なノードに紐づく → ジャンプも全件動く
- 「未リンクページ」状態が原理的に発生しない
- pageDir 共有時の重複ヒットも解消
- true orphan な .md（ノード削除後の取り残し）は検索対象外

### 実装
[notes-file-manager.ts:818-852](src/shared/notes-file-manager.ts#L818-L852):
```ts
for (outline of outFiles) {
  outData = JSON.parse(...)
  pDir = resolve(outData.pageDir or 'pages')
  for (node of outData.nodes) {
    if (!node.pageId) continue
    mdPath = join(pDir, `${pageId}.md`)
    if (!existsSync(mdPath)) continue
    searchMdFile(mdPath, ..., parentNodeText: node.text)
  }
}
```

## 確認した補足事項
- `fractal-md` skill ([claude_skills/fractal-md/scripts/fractal-md.mjs:612-613](claude_skills/fractal-md/scripts/fractal-md.mjs#L612-L613)) は `isPage: true, pageId: imported.pageId` を正しくノードに設定している。skill 側にバグはない。
- 仕様としては各 outline は `./${outlineId}/` 配下に自分のページを持つ ([notes-file-manager.ts:636](src/shared/notes-file-manager.ts#L636))。新仕様だが古い outline は default `./pages` にあり混在する可能性があり、新コードはどちらでも正しく動く。
