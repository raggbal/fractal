# Release v0.195.664

**Release date:** 2026-04-13
**Version type:** minor (new feature)
**Requirements:** requirement v8 / design v8

## Summary

File attachment support for outliner editor and MD editor. Any file type can be attached to outliner nodes (📎 icon) and embedded as links in markdown documents via drag & drop. Per-file MD footer directives (IMAGE_DIR etc.) removed in favor of VS Code settings only.

## Changes

### New Features
- **FR-1: Outliner Import Any Files** — "Import any files..." in ⋮ menu, multi-select file picker, files copied preserving original names
- **FR-2: File Node Display** — 📎 icon, editable text, child nodes supported
- **FR-3: File Node Interaction** — Click 📎 to open with OS default app. Context menu: Open File / Remove File. isPage/filePath mutually exclusive
- **FR-4: .out Schema Extension** — `filePath` field (string|null), `fileDir` top-level field. Backward compatible
- **FR-5: File Directory Settings** — `fractal.outlinerFileDir` (default `./files`). Notes mode: `{outlinerId}/files/`. Standalone outliner side panel: `{pageDir}/files/`
- **FR-6: MD File D&D** — Non-image files → copy + insert `[📎 filename](path)`. Works in standalone MD, standalone outliner side panel, and notes side panel
- **FR-7: File Link Click** — Opens with OS default app via `vscode.env.openExternal`
- **FR-8: MD File Directory Settings** — `fractal.fileDefaultDir`, `fractal.forceRelativeFilePath` via VS Code settings
- **FR-9: Cleanup Extension** — Alive set tracks `node.filePath` + `[📎]` MD links. Orphan files detected
- **FR-10: Copy/Paste** — File node copy duplicates physical file. Immediate filePath assignment on paste

### Removed
- **Per-file MD footer directives** — IMAGE_DIR, FORCE_RELATIVE_PATH, FILE_DIR, FORCE_RELATIVE_FILE_PATH all removed
- **Per-file settings button (⚙)** — Removed from sidebar for both image and file directory

### Changed
- Non-MD local file links open with OS default app (was VS Code text editor)
- Sidebar directory display simplified to single line without source label
- Side panel MD now shows word count

## Key Implementation Decisions
- `filePath` as single truthy string (not isFile+fileId) — simpler, original filename preserved
- Standard markdown link `[📎 name](path)` — compatible with other editors
- Mutual exclusion isPage/filePath — child nodes for composition
- Side panel file dir fixed to `{pageDir}/files/` — same pattern as `{pageDir}/images/`
- Per-file directives removed — low utility, high complexity

## Known Limitations
- File type-specific icons not supported (all files use 📎)
- No inline file preview
- No drag & drop directly onto outliner tree (menu import only)
