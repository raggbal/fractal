# 0.195.616

## 概要
2件の追加修正:
1. サイドパネル MD を閉じるとアウトライナのカーソルが先頭ノードに戻ってしまう
2. Markdown のリンク `[text](url)` / 画像 `![alt](url)` 内のキーワードにジャンプできない

## 課題1: サイドパネル close 時のフォーカス消失

### 症状
検索結果クリック → outliner 該当ノードへジャンプ（成功）→ サイドパネルで .md 開く → 該当箇所ジャンプ（成功）。ここまで OK。しかしサイドパネルの × や Esc で閉じると、せっかくの該当ノードフォーカスが失われ、**先頭ノードに勝手に戻ってしまう**。

### 原因
[outliner.js:4075-4080](src/webview/outliner.js#L4075-L4080) の `closeSidePanelImmediate` で:
```js
if (sidePanelOriginNodeId) {
    focusNode(sidePanelOriginNodeId);
    sidePanelOriginNodeId = null;
} else {
    focusFirstVisibleNode();   // ← これが暴発
}
```
`sidePanelOriginNodeId` は通常のノードダブルクリック経路 (`openPage` 関数) でしか設定されない。検索ジャンプ経由ではセットされないため else ブランチが発火し、`focusFirstVisibleNode()` が先頭に戻していた。

### 対策
else 節を削除。原点不明の場合は現在のフォーカスを保持する方が常に正しい（既にジャンプ済みのノードがそこにあるはず）。

## 課題2: リンク / 画像のキーワードがジャンプできない

### 症状
`[MCP Server](#11-mcp-server)` という行で `mcp` を検索すると検索結果には出るが、クリックしてもジャンプしない（または別の場所に飛ぶ）。`![image](url)` 系も全滅。

### 原因
backend [searchMdFile](src/shared/notes-file-manager.ts#L854-L884) は raw .md を split('\n') して各行を regex 検索する。`[MCP Server](#11-mcp-server)` の場合、リンクテキスト部分とURL部分の両方で `mcp` がヒットし、occurrence は 0,1 として出る。

しかし DOM レンダ後は:
- `<a href="#11-mcp-server">MCP Server</a>` — URL は **属性** になりテキストノードに出ない
- `<img src="url" alt="alt">` — alt も src も属性、テキストノードは存在しない

webview の TreeWalker (SHOW_TEXT) はテキストノードしか見ないので、occurrence と DOM 上の実ヒット数がズレる。結果、N 番目を指しても存在しない位置を指して失敗。

### 対策
backend で raw 行を **DOM 表示テキストと一致するように正規化** してから検索する。

[notes-file-manager.ts:868-884](src/shared/notes-file-manager.ts#L868-L884):
```ts
const normalized = lines[i]
    .replace(/!\[[^\]]*\]\([^)]*\)/g, '')    // 画像は完全削除
    .replace(/\[([^\]]+)\]\([^)]*\)/g, '$1') // リンクはテキストだけ残す
    .substring(0, 200);
```
これで raw 検索結果が DOM のテキストノード出現順と1対1で対応し、occurrence index がズレなくなる。画像は意図的に検索対象外（要件として alt 部分の検索は不要との合意）。
