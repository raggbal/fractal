# Release v0.195.711

**リリース日:** 2026-04-18
**バージョン種別:** patch
**対応要件:** requirement v11 / design v11
**ブランチ:** feature/v11-notes-ux-3pack

## 概要

Notes パネルの UX 改善 3 点を一括対応:
1. side panel の MD 内画像を全画面表示中 (lightbox) に ESC を押した時、lightbox のみが閉じ side panel は維持されるよう修正
2. Notes タブのフォルダ配下のファイルアイコンが親フォルダアイコンより右に配置されるようインデントを強化
3. Notes タブのフォルダ・ファイルアイコンに固定 20 色 (Tailwind 500 パレット) のカラータグを設定できる新機能

ユーザー視点で見ると、(1) は不快な操作ストレスの解消、(2) は階層構造の視認性向上、(3) は多数の `.out` を扱う際の視覚的識別性向上。

## 変更内容

### 新機能

- **FR-11-3 (NT-20): Notes タブのアイコン色付け** — Notes パネル左ペインのフォルダ・ファイル (.out) アイコンに 20 色固定パレットから色を割り当てられる。
  - 操作: 行を右クリック → "Set Color" → サブメニューに 20 swatch + "None" + "← Back"
  - パレット (Tailwind 500): red `#ef4444`, orange `#f97316`, amber `#f59e0b`, yellow `#eab308`, lime `#84cc16`, green `#22c55e`, emerald `#10b981`, teal `#14b8a6`, cyan `#06b6d4`, sky `#0ea5e9`, blue `#3b82f6`, indigo `#6366f1`, violet `#8b5cf6`, purple `#a855f7`, fuchsia `#d946ef`, pink `#ec4899`, rose `#f43f5e`, slate `#64748b`, gray `#6b7280`, zinc `#71717a`
  - 永続化: `outline.note` JSON の `items[id].color` フィールド (色名文字列)。"None" 選択時は `delete item.color` で undefined 化
  - 着色対象: アイコン SVG の `stroke` のみ (テキスト・背景は変えない)
  - 対象範囲: Notes mode の Notes タブのみ (Standalone outliner / Standalone MD には適用しない)
  - i18n: en "Set Color"/"None"、ja "色を設定"/"なし"、他 5 言語も追加。色名自体は英語のみ (tooltip 用途)

### バグ修正

- **FR-11-1: Side panel MD 画像 lightbox の ESC 動作** — side panel で MD 内の画像をダブルクリックして全画面 lightbox を開いた状態で ESC を押すと、これまでは lightbox と side panel の両方が閉じていた。修正後は lightbox のみが閉じ side panel は維持される。
  - lightbox 未表示時の ESC は従来どおり side panel を閉じる (退行なし)
  - editor.js / outliner.js 両方の ESC handler に修正を適用 (片方だけだと outliner 経由 side panel 経路で再発するため)

### 改善

- **FR-11-2: Notes タブ フォルダ配下のインデント** — `.file-panel-folder-children` の `padding-left` を 12px → 28px に拡大。子のファイル/フォルダアイコンが親フォルダアイコンより 10px 右に配置されるようになり、「フォルダの中に入っている」感が視覚的に明確になった。ネスト 2 階層以上でも累積で自然に階層感が深まる。

### 破壊的変更

なし。`outline.note` スキーマは後方互換 (`color?: string` を optional 追加、未設定の既存ファイルは引き続き正常に読み込める)。

### 非機能改善

- **セキュリティ**: webview→host 境界での入力バリデーション。`notesSetItemColor` メッセージで送信される `color` 値を `NOTES_COLOR_PALETTE` の name 配列に含まれるかでバリデーションし、不正な値 (空白入り文字列、パストラバーサル風文字列、XSS 文字列、パレット外の色名等) は `console.warn` + 早期 return で拒否。CSS class injection を防止
- **後方互換**: 既存の `outline.note` (color フィールドなし) を開いても壊れない。読み込み時 `color === undefined` を色なしとして扱う

### 主要な実装判断 (design.md / session-log.md より転記)

#### FR-11-1 ESC 処理: side panel handler 側で早期 return

選択肢:
- (A) side panel ESC handler に lightbox overlay 存在チェック追加 ✅ 採用
- (B) lightbox の escHandler で `stopImmediatePropagation()` で他 listener 抑止 ❌ 却下 (挙動が分散して追跡困難)
- (C) lightbox を capture phase で登録して bubble の側 panel より先に走らせる ❌ 却下 (phase 管理を増やすだけで利得なし)

**理由**: 既存の action-panel / command-palette のスキップパターンと完全に対称。コード変更 1 行追加のみ。document.keydown は登録順に bubble phase で発火するため、init 時に登録される側 panel handler が dblclick 時に動的登録される lightbox handler より**先に**発火する性質を利用。

#### FR-11-3 色定数の所在: 専用ファイル分離

選択肢:
- (A) `notes-color-palette.js` を新規作成 ✅ 採用
- (B) `notes-body-html.js` 冒頭埋め込み ❌ 却下 (notes-file-panel.js から参照できない)
- (C) `notes-file-panel.js` 埋め込み ❌ 却下 (CSS 生成側から参照しにくい)

**理由**: CSS 生成 (notes-body-html.js) と DOM 描画 (notes-file-panel.js) の 2 箇所から参照されるため単一真実にする。build-standalone-notes.js への copy 追加は必要。

#### FR-11-3 色選択 UI: Inline replace

選択肢:
- (A) Inline replace (Set Color クリック → 同一 menu 内で innerHTML をパレットに置換、Back で戻る) ✅ 採用
- (B) Hover submenu (右側に別 menu 要素) ❌ 却下 (位置計算コスト高)
- (C) パレット常時展開 (メニュー最下部に 20 色常時表示) ❌ 却下 (色を使わないユーザーに視覚ノイズ)

#### FR-11-3 CSS セレクタ: 直下セレクタ厳守

`.notes-item-color-{name} > .file-panel-item-icon` の**直下セレクタ**を必ず使う。子孫セレクタ (スペース) を使うとフォルダの color がサブツリー全体に伝播してしまう (子フォルダの icon まで色が変わる)。フォルダの class は wrapper ではなく **header に付与** することで、CSS の直下セレクタが正しく機能する。

#### FR-11-3 メッセージング: 2 箇所ルール (5 箇所ルールの誤適用回避)

`patterns/common/sidepanel-messaging.md` の 5 箇所ルールは「side panel editor 経由のメッセージ」が対象。今回の `notesSetItemColor` は **Notes 左パネル UI のみで発火し side panel editor を経由しない**ので 2 箇所追加 (host-bridge + message-handler) で完結する。outliner.js setupHostMessages / SidePanelHostBridge / editorProvider.ts は触らない。

#### FR-11-3 セキュリティ: webview→host 境界の allowlist 検証 (サイクル 2 で追加)

サイクル 1 のレビューで指摘された major セキュリティ問題への対応。`notes-message-handler.ts` の `notesSetItemColor` で `message.color` が `NOTES_COLOR_PALETTE` の name 配列に含まれるかを検証し、不正な値は `console.warn` + 早期 return で拒否する。class injection (例: `"red inject-class"` が複数クラスに展開される問題) を未然に防ぐ。

### 触っていない領域 (意図的)

- outliner の undo/redo スタック — 色変更は undo 対象外 (既存 outline.note 操作と一貫)
- lightbox 側の escHandler 実装 — 修正は side panel ESC handler 側のみで完結
- Standalone outliner / Standalone MD — 色付けは Notes mode のみ (Non-Goal)
- カスタムカラーピッカー UI — 固定 20 色のみ (Non-Goal)

## 既知の制限事項・残課題

### 次スプリント候補 (任意、v11 APPROVE を妨げない)

- **アクセシビリティ minor**: パレット swatch に `role="button"` / `aria-label` (色名) / `tabindex="0"` 未設定。スクリーンリーダーは「div」と読み、Tab フォーカス不可。retro 候補として記録予定

### Non-Goals (今回スコープ外、明示)

- カスタムカラーピッカー (HEX 入力 / カラーホイール) — 固定 20 色のみ
- 色によるソート / フィルタ / 検索機能
- 固定タグ色との連動
- アイコン全体の fill 着色 (stroke のみ)
- テキスト・背景の着色
- Standalone outliner / Standalone MD の色付け対応
- 色プリセットのカスタマイズ UI
- 色のテーマ連動 (ダーク/ライト自動切替) — 初版は同一色
- 色変更の undo/redo 対応 — 既存 outline.note 操作と一貫
- lightbox 自体の UI 変更 — 挙動修正のみ
