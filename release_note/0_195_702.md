# Release v0.195.702

**リリース日:** 2026-04-14
**バージョン種別:** patch
**対応要件:** requirement v10 / design v10

## 概要

MD エディタに翻訳機能（AWS Translate）を追加。さらに Wikipedia 引用リンク貼り付けのバグ修正、翻訳 UI 改善（← Back ボタン、ローディング表示、ヘッダー単行化）を含む。

## 変更内容

### 新機能
- **FR-v10: Translation** — MD 全文または選択範囲を AWS Translate で翻訳。standalone MD のツールバーと outliner side panel のヘッダーから起動。結果は readonly の side panel に表示し、`← Back` で元の MD に戻れる。翻訳中はローディングオーバーレイを表示。14 言語対応（QuickPick で選択）。デフォルト en→ja。
- **Settings**: `fractal.translateSourceLang` / `translateTargetLang` / `transAccessKeyId` / `transSecretAccessKey` / `transRegion` を追加。S3 と認証情報を独立管理できる。

### バグ修正
- **Translation: 言語選択が反映されないバグ** — outliner.js で `host.translateContent(...)` / `host.translateSelectLang(...)` が `'ja'`, `'en'` にハードコードされていた。outliner スコープに `sidePanelTranslateSourceLang` / `TargetLang` 状態を持たせ、`translateLangSelected` メッセージで更新するように修正。
- **Translation: 翻訳結果が MD テキストのまま表示されるバグ** — AWS Translate が `- [text](url)` を `-[訳] (url)` に破壊（dash 後の空白消失、`]` `(` 間の空白挿入）するのが原因。`editor-utils.js` に `normalizeTranslatedMarkdown()` を追加し、rendering 前に以下を正規化:
  - `] (url)` → `](url)` (link bracket→paren spacing)
  - `^-[` / `^*[` / `^+[` → `- [` (bullet before link)
  - `^-word` → `- word` (bullet before text, hr `---` は除外)
  - `^1.text` → `1. text` (ordered list)
  - `^#text` → `# text` (heading)
- **Wikipedia 引用リンク貼り付けのバグ** — Wikipedia の `[40]` 引用をコピペすると `[[40]](url)` となり、`[40]` 全体がリンクになっていた。以下 2 点を修正:
  - Turndown `normalizeLink` ルール: link text が `[...]` / `\[...\]` パターンの場合、外側ブラケットをリンク外に出す (`[[text](url)]` 形式で出力)
  - `parseMarkdownLinks`: `[[text](url)]` wrapper パターンを検出し、外側 `[` をスキップして内側を正しくリンクとして解析

### 非機能改善
- Translation 言語ボタン表示を単行化 (`en → ja` → `ja` のみ。source → target は tooltip で確認可能)。ヘッダー高さが縦に広がる問題を解消。

### 主要な実装判断
- **AWS Translate は AWS CLI 経由** (SDK 不使用): `@aws/lambda-invoke-store` 依存問題で VSCode 拡張環境では SDK が動作しないため、既存 `notes-s3-sync` と同じ CLI 方式で統一。
- **翻訳結果パネルは readonly な EditorInstance 再利用**: 独立 UI を作らず既存の side panel + `readonly: true` オプションで実現。`openTranslationPanel` (standalone) / `showTranslationInSidePanel` (outliner) の 2 経路。
- **MD 正規化はクライアント側 (webview)**: AWS の返答を信用せず、rendering 直前に syntax を修復。サーバー側で処理すると DoD の test が難しいため。
- **Wikipedia 修正の 2 段構え**: turndown 出力を `[[text](url)]` に変えるだけでは MD パーサが外側 `[` を貪欲に消費してしまうため、parser 側にも wrapper 検出を追加。両側で整合性確保。
