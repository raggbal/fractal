# Release v0.195.708

**リリース日:** 2026-04-14
**バージョン種別:** patch
**対応要件:** Fix モード（`.harness/investigation/investigation-20260414-142721.md`）

## 概要

サイドパネルのアウトライン（目次）で h3〜h6 が表示されないバグを修正。`# Title` の直後に `### Sub` が来る非階層的な見出し構造でも、全ての見出しが正しく拾われるようになった。

## 変更内容

### バグ修正

- **Side panel TOC が h3-h6 を取りこぼす問題を修正** — side panel の TOC 抽出に使われる正規表現が `/^(#{1,2})\s+(.+)$/` となっており、h1 / h2 しかマッチしていなかった。`/^(#{1,6})\s+(.+)$/` に変更し、h1-h6 を全て抽出するようにした。非階層（h1 → h3 のような h2 スキップ）でも全見出しが表示される。
  - Standalone editor は `editor.querySelectorAll('h1,h2,h3,h4,h5,h6')` で DOM から拾うため影響なし（現象は side panel 文脈のみ）。
  - CSS (`.side-panel-toc-item[data-level="3"..."6"]`) は既に h3-h6 用の padding/font-size を定義済みだったため、正規表現修正だけで表示は正しく整う。

### 非機能改善

- **重複ロジックを `src/shared/toc-utils.ts` に集約** — 同一の TOC 抽出ロジックが 3 ファイルに重複していた。ユニットテストのために vscode 依存を排除する必要があったため、pure function として切り出した（`extractToc(markdown): TocItem[]`）。`sidePanelManager.ts` は内部でこの関数に委譲する。
- **ユニットテスト `test/specs/unit-sidepanel-toc.spec.ts` を追加** — h1-h6 全抽出 / h1→h3 非階層 / h1+h2 リグレッション / コードブロック内の `#` を無視 / CJK アンカー生成、の 5 ケース。

### 主要な実装判断

- **正規表現修正だけでなく `toc-utils.ts` への抽出を行った理由**: 同一ロジックが 3 ファイルに存在し、片方だけ修正されるリグレッションリスクが高い。かつ `sidePanelManager.ts` は `vscode` モジュールをインポートしているため Playwright からの単体実行が困難。pure function として分離することで、今後 h* 抽出を変更する際に 1 箇所修正で済み、かつ単体テストで振る舞いを固定できる。
- **階層再構築はしない（フラット配列 + data-level 属性で表現）**: 「h1 → h3」を「h2 相当に昇格」「h1 配下にぶら下げる」等の案もあったが、ユーザー側の階層意図を勝手に歪めるため採用せず。CSS が level ごとの padding/font-size を持つので、見出しレベルは素直にそのまま表示する。
