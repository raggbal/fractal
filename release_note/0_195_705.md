# Release v0.195.705

**リリース日:** 2026-04-14
**バージョン種別:** patch
**対応要件:** requirement v10 / design v10（翻訳機能の UX 改善フォローアップ）

## 概要

v0.195.702 で導入した翻訳機能の UX 問題 2 件を修正。side panel の Expand ボタン位置も使いやすく改善。

## 変更内容

### バグ修正
- **翻訳結果パネルを閉じた後に再度開くとヘッダーが壊れる** — outliner の side panel で翻訳結果を表示中に Esc / パネル外クリック / × で閉じ、次回パネルを開くと `← Back` ボタンだけの状態で再利用され、undo/redo/translate 等のデフォルトボタン群が復元されなかった。修正:
  - `showTranslationInSidePanel` で `closeSidePanelImmediate()` が `sidePanelPreTranslationState` を null 化する前に、元ヘッダー HTML (`actionsHtml`) を保存 → 閉じた後に state を復元
  - `closeSidePanel` / `closeSidePanelImmediate` の両方で、閉じる直前に `restoreHeaderActionsFromTranslation()` を呼び出して action ボタン HTML を元に戻す

### 非機能改善
- **Expand ボタンを左端に移動** — side panel のヘッダーで Expand (拡大) ボタンが右側にあり、パネル幅が狭いと隠れて使えなくなる問題。`.side-panel-header-actions` の先頭（Undo の左）に配置変更して常に見える位置にした。

## 実装詳細

### 変更ファイル一覧

| ファイル | 変更種別 | 内容 |
|---|---|---|
| `package.json` | 修正 | version 0.195.702 → 0.195.705 |
| `src/shared/editor-body-html.js` | 修正 | Expand ボタンを `.side-panel-header-actions` の先頭に移動（`side-panel-header-btn` クラスも付与しつつ `.side-panel-expand` クラスは保持） |
| `src/webview/outliner.js` | 修正 | `restoreHeaderActionsFromTranslation` 関数を追加、`closeSidePanel`/`closeSidePanelImmediate` 両方で呼び出し。`showTranslationInSidePanel` で closeSidePanelImmediate の前に actionsHtml を保存し state を復元 |
| `CHANGELOG.md` | 修正 | v0.195.705 エントリ追加 |

### 主要な実装判断
- **actionsHtml は `sidePanelPreTranslationState` に保存**: 新規のグローバル変数を増やさず既存の「翻訳前に戻す」state に保存。close handler から一元的に参照可能。
- **Expand ボタンにクラス併用**: `.side-panel-expand` の既存スタイル・クリックハンドラを保ったまま `.side-panel-header-btn` を追加することで、action ボタン群と同じ行内レイアウトに組み込む。
