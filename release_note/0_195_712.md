# Release v0.195.712

**リリース日:** 2026-04-19
**バージョン種別:** patch (monotonic patch bump)
**対応要件:** requirement v12 / design v12 (OL-30: Outliner Drag & Drop ファイル import)

## 概要

Outliner editor (Notes mode + standalone) に OS からのファイルドラッグ&ドロップ (D&D) による import 機能を追加した。既存の ⋮ メニュー経由の「Import .md files...」「Import any files...」はそのまま残し、D&D は追加の入力経路として並列で動作する。v8 で設定された「Outliner への D&D ファイル (v8: メニュー import のみ)」という Non-Goal は v12 で lift された。

## 変更内容

### 新機能

- **OL-30: Outliner Drag & Drop ファイル import**
  - **対象 UI 領域**: outliner ツリーエリアのみ (side panel / toolbar / header / 固定タグバー / リサイズハンドルは対象外)
  - **ドロップ位置判定**: 既存ノード reorder D&D (OL-12) と同じ 25%/50%/25% 規則
    - ノード行上 25% → before sibling として挿入
    - 中央 50% → child として挿入 (collapsed 親は expand)
    - 下 25% → after sibling として挿入
    - 空エリア/空ツリー → ルートレベル最後尾
  - **ファイル種別自動振り分け**:
    - `.md` → OL-19A 相当 (H1抽出 + ページノード + pageId)
    - 画像 (.png/.jpg/.jpeg/.gif/.webp/.svg/.bmp) → OL-15 相当 (inline image + 60x60 サムネイル + images 配列)
    - その他 → OL-19B 相当 (file directory コピー + filePath 添付ノード)
  - **複数ファイル同時ドロップ**: `dataTransfer.files` 順序維持、各ファイル個別振り分け、1 D&D = 1 undo step
  - **既存ノード reorder との分岐**: `e.dataTransfer.types.includes('Files')` で判定
  - **フォルダ拒否**: `webkitGetAsEntry().isDirectory` で検出 → 警告通知
  - **大容量拒否**: 50MB 超は webview 側で拒否 + `notifyDropFileTooLarge` 通知
  - **視覚フィードバック**: ツリー全体に dashed outline (`.outliner-tree-drop-zone-active`) + 既存位置インジケーター線
  - **Notes mode + standalone 両対応**: 同じ `src/webview/outliner.js` が両モードで動作

### 変更 (既存機能の維持)

- **OL-19B (v8 → v12)**: ⋮ メニュー「Import any files...」は廃止せず存続。D&D と内部 core (`importFilesCore`) を共有
- **OL-19A**: ⋮ メニュー「Import .md files...」も存続。D&D と `importMdFilesCore` を共有
- **OL-15**: Cmd+V image paste は既存の独立パスで動作 (D&D は別経路で既存に影響なし)
- **OL-12**: ノード reorder D&D は `e.dataTransfer.types` 判定により完全に共存

### 破壊的変更

なし。既存の `importFiles()` / `importMdFiles()` の外部シグネチャは完全に不変 (内部で core 関数に委譲する thin wrapper に整理)。

### 非機能改善

- **パフォーマンス**: 10MB 単一ファイル drop < 200ms、20 ファイル同時 drop < 1s (NFR-12-1)
- **セキュリティ**: path traversal 防御を `importFilesCore` 冒頭で強化 (`../`, 絶対パス, 埋込 `..` を拒否)
- **コード再利用**: 既存 `saveOutlinerImage` ハンドラ / `importFiles` / `importMdFiles` の重複実装なし、全て内部 core 経由で統一

### 主要な実装判断

- **1 D&D = 1 saveSnapshot**: 複数ファイル混在 (md + image + file) でも単一 snapshot で undo/redo 整合性を保つ
- **単一メッセージ `dropFilesImport`**: webview で kind を確定してから 1 メッセージで送信。3 メッセージ分割案は 1 snapshot 制約と矛盾するため却下
- **画像も新規ノード作成 + addImage**: 既存ノードへの破壊的 inline 注入は避け、D&D 位置で新ノードを作って addImage する設計
- **`saveOutlinerImage` ハンドラは内部呼出しない**: batch 文脈では `outlinerImageSaved` メッセージが per-image で発火して混乱するため、`drop-import.ts` 内部で base64 decode + `fs.writeFileSync` を直接実行
- **core 抽出パターン**: `importFiles`/`importMdFiles` の外部 API を変更せず、内部 core 関数を共通化。DRY とテスト集約を両立
- **D&D 経路での相対画像参照スキップ**: webview の File API は source directory を提供しないため、dropped `.md` 内の相対画像パスはスキップ (新 option `skipRelativeImages: true`)。メニュー版は file path から source directory が取れるため既存通り動作

## 既知の制限事項・残課題

- D&D で取り込む `.md` の**相対画像参照はスキップ**される (broken link として .md に残る)。webview セキュリティ制約による不可避の仕様。メニュー版では従来通り画像コピー + パス書換が行われる
- 50MB 超ファイルは webview で拒否 (v12 では非対応)。将来スプリントで非同期化検討可
- フォルダの D&D import は未対応 (v12 では拒否 + 通知のみ)。再帰 import は事故リスクが高いため意図的にスコープ外
- リモート URL (`http(s)://`) の D&D は未対応 (v12: ローカルファイルのみ)
