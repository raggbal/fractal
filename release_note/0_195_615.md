# 0.195.615

## 概要
Notes 検索の Markdown 結果クリック時、4段階のジャンプ動作のうち③（サイドパネルで .md を開く）以外がほぼ全滅していた問題の根本修正。

## 課題
本来の動作:
1. ① 該当 .out (outline) を開く
2. ② その outline の該当ノードへスクロール＆ハイライト
3. ③ サイドパネルで .md を開く
4. ④ サイドパネル内の該当キーワード位置へスクロール＆ハイライト

実際は ③ だけが動き、①②④ は全滅。

## 原因
2箇所の独立したバグが重なっていた。

### 原因1: ②ノードジャンプ失敗 — `jumpToNodeId` 付与漏れ
[notes-message-handler.ts:649-653](src/shared/notes-message-handler.ts#L649-L653) の `notesJumpToMdPage` ハンドラは `pageId → nodeId` の逆引きまではしていたが、`updateData` メッセージに `jumpToNodeId` を付けずに送信していた。outliner.js は `updateData.jumpToNodeId` が存在するときだけ `jumpToAndHighlightNode` を呼ぶ仕様 ([outliner.js:4500-4503](src/webview/outliner.js#L4500-L4503))。

比較対象の `notesJumpToNode` ハンドラ ([notes-message-handler.ts:606-623](src/shared/notes-message-handler.ts#L606-L623)) は `jumpToNodeId: message.nodeId` を正しく付けている。

### 原因2: ④キーワードスクロール失敗 — メッセージ経路断絶
notes mode の webview は outliner.js が動いており、サイドパネル editor は同じ webview 内で生成された別 EditorInstance。`notesEditorProvider` から送られた `scrollToText` メッセージはまず outliner.js のリスナで受け、そこから `sidePanelHostBridge._sendMessage` でサイドパネル EditorInstance のリスナへリレーする必要がある。

[outliner.js:4624](src/webview/outliner.js#L4624) には `scrollToLine` のリレーは存在したが、**`scrollToText` のリレーが存在しなかった**ため、editor.js の `scrollToText` ハンドラが呼ばれることがなかった。

## 対策

### 修正1
[notes-message-handler.ts:649-654](src/shared/notes-message-handler.ts#L649-L654) で `updateData` に `jumpToNodeId: pageNodeId` を追加。

### 修正2
[outliner.js:4630付近](src/webview/outliner.js#L4630) に `case 'scrollToText'` を追加し、`sidePanelHostBridge._sendMessage({type:'scrollToText', text, occurrence})` でリレー。

## 検証経路
```
検索結果クリック
  ↓ jumpToMdPage(query, occurrence)
notes-message-handler.notesJumpToMdPage
  ├→ updateData { jumpToNodeId: pageNodeId }   ← 修正1
  │    ↓ outliner.js → jumpToAndHighlightNode  → ② OK
  └→ platform.openPageInSidePanel
        ├→ sidePanel.openFile                  → ③ OK (元から動作)
        └→ panel.webview.postMessage('scrollToText')
              ↓ outliner.js scrollToText リレー ← 修正2
                ↓ sidePanelHostBridge._sendMessage
                  ↓ editor.js scrollToText      → ④ OK
```
