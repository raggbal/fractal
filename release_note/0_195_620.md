# 0.195.620

## 概要
Outliner の2件の操作系バグ修正:
1. 複数行選択して Tab を押すと、兄弟の2番目以降が余計にインデントされる
2. 子を持つノードで Enter を押すと、新規ノードが兄弟ではなく子として挿入される

## バグ1: 複数選択 Tab で兄弟が余計にインデント

### 課題
```
- w
- a
  - b1
  - b2
  - b3
```
a〜b3 を複数選択して Tab → 期待は全行が1レベル一律 indent。
実際:
```
- w
  - a
    - b1
      - b2  ← 余計
      - b3  ← 余計
```

### 原因
[outliner.js:2086-2094](src/webview/outliner.js#L2086-L2094) の Tab ハンドラは全選択ノードを順次 `model.indentNode` していた。`indentNode` は「前の兄弟の子の末尾に移動」する仕様 ([outliner-model.js:264](src/webview/outliner-model.js#L264))。b1 が a の子になった直後、b2 の「前の兄弟」は b1（すでに a の子）→ b2 は b1 の子として誤ってネストされる。

0.195.607 で Shift+Tab 側 (outdent) では「選択ルート（親が選択外のノード）のみ処理し、子は親に追従」という修正が入っていたが、Tab 側 (indent) に対称修正が入っておらず片側実装のままになっていた。

### 対策

**設計**
Shift+Tab と同じく「選択ルートのみ indent」方式に統一する。子は親ノードの移動に自動で追従するため、相対階層が保たれる。

**実装** ([outliner.js:2086-2105](src/webview/outliner.js#L2086-L2105))
```js
} else {
    // Tab: 選択ルート（親が選択外のノード）のみindent。子は親に追従
    var indentRootIds = [];
    for (var si2 = 0; si2 < sortedIds.length; si2++) {
        var sn2 = model.getNode(sortedIds[si2]);
        if (!sn2) { continue; }
        if (!sn2.parentId || !selectedNodeIds.has(sn2.parentId)) {
            indentRootIds.push(sortedIds[si2]);
        }
    }
    var topRootNode = indentRootIds.length > 0 ? model.getNode(indentRootIds[0]) : null;
    var topRootInfo = topRootNode ? model._getSiblingInfo(indentRootIds[0]) : null;
    if (topRootInfo && topRootInfo.index > 0) {
        for (var ti = 0; ti < indentRootIds.length; ti++) {
            if (model.indentNode(indentRootIds[ti])) { anyMoved = true; }
        }
    }
}
```

## バグ2: 子持ちノードで Enter すると子として挿入される（初回修正）

### 課題
```
- a|          (カーソルが a の末尾、a には子 b)
  - b
```
Enter を押すと:
```
- a
  - |        ← 新ノードが a の子として入ってしまう
  - b
```
期待: 常に兄弟として挿入されるべき（Dynalist 風の「子先頭挿入」は要件にない）。

### 原因
[outliner.js:2455-2460](src/webview/outliner.js#L2455-L2460) の `handleEnter` が次のような分岐を持っていた:
```js
if (node.children.length > 0 && !node.collapsed) {
    newNode = model.addNodeAtStart(node.id, afterText);   // ← 子として先頭追加
} else {
    newNode = model.addNode(node.parentId, node.id, afterText);
}
```

### 対策

**設計**
- Enter は「常に兄弟として挿入」をデフォルト動作とする
- カーソル先頭 (`offset===0`) でテキストがある場合は、現ノード(テキスト+子)を維持したまま空ノードを前に挿入する（`|a` で Enter → 空行が `a` の上に入る自然な挙動）
- 子として挿入したい場合は既存の Shift+Enter (`handleShiftEnter`) を使う（既に実装済み）

**実装** ([outliner.js:2461-2485](src/webview/outliner.js#L2461-L2485))
```js
var newNode;
if (offset === 0 && text.length > 0) {
    // 現ノードは変更せず、前に空ノードを挿入
    var siblings = node.parentId
        ? (model.getNode(node.parentId) || {}).children || []
        : model.rootIds;
    var idxInSiblings = siblings.indexOf(node.id);
    if (idxInSiblings <= 0) {
        newNode = model.addNodeAtStart(node.parentId, '');
    } else {
        newNode = model.addNode(node.parentId, siblings[idxInSiblings - 1], '');
    }
} else {
    model.updateText(node.id, beforeText);
    newNode = model.addNode(node.parentId, node.id, afterText);
}
```

### 既知の残課題
この時点では `a|` + child `b` → Enter の結果が:
```
- a
  - b
- |
```
となり、新ノードが `b` の下に配置される（子は現ノードに残る）。視覚的に「すぐ下の行」に現れないため、0.195.621 で追加修正する。
