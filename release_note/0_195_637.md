# 0.195.637

## 概要
Markdown の画像パスや URL に `()` (半角括弧) が含まれていると、リンク/画像が壊れる問題を全面的に修正。balanced paren parser を新規作成し、editor / outliner / import / paste / skill の全経路で統一的に対応した。

## 課題

### 課題 1: 画像パスに `()` が含まれると画像が壊れる
ファイル名に `(v2)` `(copy)` `(1)` 等を含む画像 (`images/photo_(v2).png`) が、Fractal Editor / Outliner / import / paste の全経路で壊れていた。

### 課題 2: URL に `()` が含まれるとリンクが壊れる
Wikipedia 系 URL `https://en.wikipedia.org/wiki/Foo_(disambiguation)` 等が最初の `)` で切れ、リンクが途中までしかクリックできなかった。

### 課題 3: Outliner の URL ペースト時のリンク化が壊れる
`https://en.wikipedia.org/wiki/Foo_(bar)` を Outliner にペーストすると `[url](url)` 生成後の Markdown link parser が `)` で壊れ、表示が崩れていた。

## 原因
全箇所 (editor.js, outliner.js, markdown-import.ts, markdown-image-utils.ts, fractal-md.mjs) で `\(([^)]+)\)` 系の正規表現を使っており、URL/パス内の `)` が「閉じカッコ」として誤マッチしていた。

## 対策

### 設計
balanced paren parser を新規作成し、`(` `)` のネスト深さをカウントして**対応する閉じカッコ**を正確に探す方式に統一。正規表現ベースから手書きスキャナに切り替えることで、任意深さのネスト `((a)(b))` にも対応。全角括弧 `（）` は文字として扱う (半角のみカウント)。
