# Release v0.195.647

**Release date:** 2026-04-12
**Version type:** minor (new feature)
**Requirements:** requirement v8 / design v8

## Summary

File attachment support for both outliner editor and MD editor. Any file type can be attached to outliner nodes (📎 icon) and embedded as links in markdown documents via drag & drop.

## Changes

### New Features

- **FR-1: Outliner Import Any Files** — "Import any files..." menu in ⋮ dropdown. Multi-select file picker, files copied to configurable directory preserving original names with collision suffix.
- **FR-2: File Node Display** — 📎 icon on file nodes, editable text, child nodes supported.
- **FR-3: File Node Interaction** — Click 📎 to open with OS default app. Context menu: "Open File" / "Remove File". isPage and filePath mutually exclusive.
- **FR-4: .out Schema Extension** — `filePath` field (string|null) on nodes, `fileDir` on top-level. Backward compatible.
- **FR-5: File Directory Settings** — `fractal.outlinerFileDir` (default `./files`). Notes mode: `{id}/files/`.
- **FR-6: MD File Drag & Drop** — Non-image files → copy to file dir + insert `[📎 filename](path)`. Image files unchanged.
- **FR-7: File Link Display & Interaction** — 📎 links rendered in HTML view, click opens with OS default app.
- **FR-8: MD File Directory Settings** — `fractal.fileDefaultDir`, `fractal.forceRelativeFilePath`, `FILE_DIR`/`FORCE_RELATIVE_FILE_PATH` footer directives, sidebar settings UI.
- **FR-9: Cleanup Extension** — Alive set tracks `node.filePath` + `[📎](path)` MD links. Orphan files detected with `$(file)` icon in QuickPick.
- **FR-10: Copy/Paste Asset Handling** — File node copy/paste duplicates physical file. Same pattern as page/image assets.

### Changed

- Non-MD local file links in MD editor now open with OS default app (`vscode.env.openExternal`) instead of VSCode text editor (`vscode.open`).

## Known Limitations
- File type-specific icons not supported (all files use 📎)
- No inline file preview in outliner or MD editor
- No drag & drop directly onto outliner tree (menu import only)
- Standalone MD editor file D&D settings are independent from outliner file settings
