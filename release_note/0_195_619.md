# 0.195.619

## 概要
Markdown 検索結果のヘッダー表示で、ノードテキストが空のときの表示をユーザーに分かる形にフォールバック。

## 課題
0.195.618 で「未リンクページ」表示は撲滅したが、ノードに紐づいているがそのノードの `text` が空のケース（画像中心ページ等）では `OutlineTitle / ${pageId先頭}` のような UUID 風表示にフォールバックしていて何のページか分からない。

## 対策

### 設計
表示ラベルを次のフォールバック順で決定:
1. `node.text` (trim 後)
2. .md ファイル先頭20行から最初の見出し (`# ... ###### ...`)
3. `pageId` の先頭8文字

### 実装
[notes-file-manager.ts:830-849](src/shared/notes-file-manager.ts#L830-L849):
```ts
let label = (nn.text || '').trim();
if (!label) {
    try {
        const mdHead = fs.readFileSync(mdPath, 'utf8').split('\n').slice(0, 20);
        for (const ln of mdHead) {
            const hm = ln.match(/^\s{0,3}#{1,6}\s+(.+?)\s*#*\s*$/);
            if (hm) { label = hm[1].trim(); break; }
        }
    } catch { /* skip */ }
}
if (!label) label = pageId.substring(0, 8);
const displayTitle = `${outTitle} / ${label}`;
```

ファイル先頭20行を読むため僅かに I/O コストが増えるが、空テキストノードに限った fallback パスなので影響は微小。
