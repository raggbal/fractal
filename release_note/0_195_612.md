Notes Editor の検索機能の不具合修正と UI 改善。

## 修正1: 検索結果クリック時にノードへ正しくジャンプしない

### 課題
検索結果をクリックしても、目的のノードに正しくスクロールしないケースがある。
特に画像のみで構成されたノードや、画像直下のノード（例: 「ITX for MCP Partner」）でジャンプしても画面の中央に該当ノードが来ない。

### 原因
`jumpToAndHighlightNode` ([src/webview/outliner.js:3372-3402](src/webview/outliner.js#L3372-L3402)) は次の順で処理していた:

1. `el.scrollIntoView({ block: 'center' })` で対象ノードを画面中央へ
2. ハイライトクラスを付与
3. `focusNode(nodeId)` を呼び出し、内部の `<input>` 要素に `focus()`

問題は (3)。ブラウザは `focus()` を受けた要素を可視化するために**自動的にスクロール**するため、せっかく (1) で中央に置いたスクロール位置が打ち消される。テキストノードでは差が小さいが、画像を含む高さの大きいノードでは、フォーカス対象（編集用 input）と画像本体が縦に離れているため、画像が画面外に押し出されてしまう。

### 対策
**設計:** `focusNode` のグローバル動作は変更しない（通常のキーボード移動などでは scrollIntoView 副作用がむしろ望ましい）。jump 経路だけでフォーカス時の自動スクロールを抑止する。

**実装:** [src/webview/outliner.js:3393-3410](src/webview/outliner.js#L3393-L3410)
- `focusNode(nodeId)` の呼び出しを削除し、jumpToAndHighlightNode 内に直接 `textEl.focus({ preventScroll: true })` を使うフォーカス処理をインライン化
- 古いブラウザの保険として try/catch でフォールバック
- 要素が DOM に存在しなかった場合は従来の `focusNode(nodeId)` にフォールバック

## 修正2: 検索結果のセクション分けが無く混在して見える

### 課題
Outliner (.out) の検索結果と Markdown (.md) の検索結果が同じリストに混在しており、どちらの結果か判別しづらい。

### 対策
**設計:** 結果リストを「Outlinerの検索結果」「Markdownの検索結果」の2セクションに常時分割。`onSearchPartial` でストリーミング到着する結果を `fileType` で振り分ける。結果0のセクションは非表示。

**実装:**
- [src/shared/notes-file-panel.js:682-755](src/shared/notes-file-panel.js#L682-L755): `onSearchStart` で空の2セクションを生成。`onSearchPartial` でファイルタイプに応じて該当セクションへ append。各セクションタイトルに件数をリアルタイム反映。
- [src/shared/notes-body-html.js](src/shared/notes-body-html.js): `.file-panel-search-section-title` スタイル追加。

## 修正3: Markdown 検索結果に表示される ID が分かりづらい

### 課題
Markdown 検索結果のヘッダーには `pageId.md`（ハッシュ風の ID）が表示されており、どのノードに紐づいたページか直感的に分からない。

### 対策
**設計:** Markdown ページの検索結果には、ページが紐づいているノード名を表示する。表示形式は階層表記 `OutlineTitle / 親ノード名`。

**実装:** [src/shared/notes-file-manager.ts:818-848](src/shared/notes-file-manager.ts#L818-L848)
- pages 検索ループ内で `pageId → ノード名` の逆引きマップを構築（`outData.nodes` の `pageId` プロパティから）
- `searchMdFile` 呼び出し時の `fileTitle` を `${outTitle} / ${nodeText}` 形式に変更
- ノード名取得に失敗した場合のみ従来通り `${outTitle} / ${pm}` にフォールバック
- `SearchResult` インターフェースに `parentNodeText?: string` を追加（将来の表示拡張用）

## 修正4: 検索結果ヘッダーが太字のみで判別しづらい

### 課題
Outliner / Markdown の各ファイルヘッダーは `font-weight: 600` で太字にされているだけで、種類の違いが視覚的に区別できない。

### 対策
**設計:** ファイルタイプ別に色を変える。VSCode テーマトークンを使ってライト/ダーク両対応。

**実装:** [src/shared/notes-body-html.js](src/shared/notes-body-html.js)
- セクションタイトル: `var(--vscode-textLink-foreground)`
- Outliner ファイルヘッダー: `var(--vscode-symbolIcon-classForeground)`
- Markdown ファイルヘッダー: `var(--vscode-symbolIcon-fileForeground)`（`.is-md` クラス）
- [src/shared/notes-file-panel.js](src/shared/notes-file-panel.js): Markdown 結果のヘッダー要素に `is-md` クラスを付与

## テスト方法
1. 既存の outliner ファイルを開き、検索パネルを開く（Notes タブから検索アイコン）
2. 任意のキーワードで検索し、結果が「Outlinerの検索結果 (n)」「Markdownの検索結果 (n)」の2セクションに分かれて表示されることを確認
3. 各セクションのタイトル / ファイルヘッダーが色付きで表示されることを確認
4. Markdown 結果のファイルヘッダーが `OutlineTitle / 親ノード名` の形式で表示されることを確認
5. 画像のみのノード（例: 「ITX for MCP Partner」）の検索結果をクリックし、ノードが画面中央にスクロールされることを確認
