# Release v0.195.722

**リリース日:** 2026-04-26
**バージョン種別:** patch (機能追加だがデフォルトで既存挙動に影響しない backwards-compatible 変更)
**対応要件:** v15 (MD-44) — Sprint `20260426-095402-toolbar-translate-toggle`

## ハイライト

1. **新設定 `fractal.showTranslateButtons` (default OFF)** — 翻訳ボタンの表示を制御。既存ユーザーへの影響あり (側 panel header の翻訳ボタンも非表示になる、ON で復元可)
2. **standalone toolbar に翻訳ボタンを追加** — toolbar-inner の最左 (inline group の直前) に translateLang + translate
3. **standalone toolbar 経由の翻訳結果は editor 自身に表示** — side panel スライドオーバーではなく、現在の editor を一時翻訳ビューに置換 + ← Back で完全復元

## 詳細な変更内容

### Added

#### 新設定 `fractal.showTranslateButtons` (FR-TRTOG-1)
- `boolean`, default `false`
- ON 時: standalone toolbar (toolbar-inner 内の inline group の直前) と side panel header の両方に translate / translateLang ボタンを表示
- OFF 時: 両方とも非表示
- VSCode 設定 UI で `Fractal: Show Translate Buttons` トグルが表示される
- 設定変更で webview reload 経由で即時反映 (既存 `onDidChangeConfiguration` パイプライン)
- description: "Show translate buttons in the editor toolbar (standalone) and side panel header. When off, translation can still be triggered via the 'fractal.translate' command (Cmd+/)."

#### standalone editor toolbar への translate group 追加 (FR-TRTOG-2)
- `editor-body-html.js` の `<div class="toolbar-inner">` 先頭 (= scroll 内最左、inline group の直前) に新規 toolbar-group を挿入
- 内部に `<button data-action="translateLang">ja → en</button>` + `<button data-action="translate"></button>`
- 既存 `host.translateContent` / `host.translateSelectLang` 経路を再利用 (機能本体 MD-42 は不変)
- side panel header の既存 translate ボタンは維持

#### standalone editor 内 inline 翻訳ビュー (FR-TRTOG-6)
- standalone toolbar の translate ボタンから来た翻訳結果は **side panel を一切使わず**、現在の editor を一時的に翻訳ビューに置換
- 元 `editor.innerHTML` (or `sourceEditor.value`) / モード / `contentEditable` / `scrollTop` / `hasUserEdited` を `translationSavedState` に保存
- 編集差分があれば事前に disk へ force-flush (翻訳ビュー中の sync 抑止のため)
- `editor.contentEditable = 'false'` で readonly 化、上部に sticky な `.fractal-translation-header` (← Back / `Translation (src → tgt)` / Copy)
- ← Back で内容・モード・編集状態を完全復元 + 保留中の cross-edit を `applyQueuedExternalChange()` で再適用 (NT-14 維持)

### Changed

#### デフォルト OFF — 既存ユーザーへの影響あり
- 本リリース後、設定 `fractal.showTranslateButtons` を明示的に ON にしない限り、**side panel header の翻訳ボタンも非表示になる**
- 翻訳機能自体は不変。command palette → `Fractal: Translate` (`fractal.translate`) で従来通り起動可能
- → 翻訳ボタンが消えたユーザーは `Cmd+,` 設定で `fractal.showTranslateButtons` を ON に切替

#### 翻訳結果の表示先 (standalone 経由)
- 従来 (v0.195.718 以前): side panel に readonly EditorInstance を slide-over (右半分を覆う)
- 新規 (v0.195.722): standalone editor 自身を一時的に翻訳ビューに置換 (元の editor の場所に表示される)
- outliner page → side panel の翻訳経路 (`outliner.js showTranslationInSidePanel`) は不変

### Fixed

#### 翻訳ビュー中の disk 上書きを防止 (NFR-TRTOG-5)
- 翻訳ビュー中に他アプリへ Cmd+Tab で切替 → 戻ってきた時、blur / visibility hidden flush で `editor.innerHTML` (= 翻訳結果) を `htmlToMarkdown()` で markdown 化 → `host.syncContent()` で disk に書き込んでしまう経路を遮断
- `translationViewActive` フラグで以下 4 箇所を early return:
  - `editor.addEventListener('blur', ...)` 全体
  - `sourceEditor.addEventListener('blur', ...)` 全体
  - `_handleVisibilityChange` 全体
  - `applyQueuedExternalChange` 内 (queue は保持、Back 時に再適用)
- `editor.contentEditable = 'false'` で input event は自動停止 (debouncedSync 経路は触らずとも止まる)
- `hasUserEdited = false` 設定の二重ガード

#### 翻訳ヘッダ label の文字色
- `.fractal-translation-label` の color を `--blockquote-color` → `--text-color` に変更
- ライト/ダーク両 theme で本文と同色 (反転して白く見える問題を解消)
