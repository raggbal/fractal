# Release v0.195.710

**リリース日:** 2026-04-15
**バージョン種別:** minor（新機能追加、後方互換あり）
**対応要件:** チャットベース実装（`/sync` 未実行 — v11 として artifact 化は次スプリントで対応予定）

## 概要

Outliner エディタに、MD エディタと同等の**テキスト検索 & 置換機能**を追加。右上にフロートする検索/置換ボックス（Cmd+F）で、現在のスコープ内のノード本文（text）とサブテキスト（subtext）を横断検索・置換できる。MD エディタと UI・キーバインドを揃えた。

既存の Cmd+F（ヘッダフィルタ検索にフォーカス）は **Cmd+Shift+F** に移設。ユーザー指示による仕様変更。

## 変更内容

### 新機能

- **テキスト検索 & 置換ボックス (Cmd+F)** — MD エディタと同じデザイン・機能の検索置換ボックスを outliner の右上に表示。
  - 検索入力 / N/M マッチカウンタ / 前後ナビゲーション (▲▼) / 置換行トグル / 閉じるボタン
  - 置換入力 / Replace / Replace All
  - オプション: 大文字小文字区別 (Aa) / 単語単位 (Ab|) / 正規表現 (.*)
  - Enter で次マッチ、Shift+Enter で前マッチ、Esc でボックスを閉じる
- **置換行トグル (Cmd+H)** — MD エディタと同様、Cmd+H で置換行を展開した状態でボックスを開く
- **検索対象** — 現在のスコープ（`scopeIn` 中ならそのサブツリー、なければ全ノード）の `node.text` と `node.subtext` の両方
- **インライン書式の維持** — `**foo**` の `foo` を検索して `bar` に置換すると、source が `**bar**` になるため太字が保持される。`renderedOffsetToSource` でレンダリング後オフセット → source オフセットへ変換してから model を更新するアルゴリズム
- **折畳ノード内マッチ** — 検索結果件数には含まれるが、該当ノードはツリー上では折り畳まれたまま。Next/Prev でジャンプした時のみ祖先チェーンを自動展開（`ensureNodeExpanded`）してスクロール
- **Replace All は 1 undo** — `(nodeId, field)` でグループ化し `sourceStart` 降順で一括書換、`saveSnapshot()` は最初に 1 回のみ
- **ハイライト** — `.outliner-search-hit`（黄）/ `.outliner-search-hit-current`（オレンジ）。`TreeWalker` + `splitText` で、インライン書式要素（`<strong>`, `<em>`, `<a>`, `<code>`, タグ span 等）をまたぐマッチも正しく分割してラップ。MD エディタの `range.surroundContents()` より堅牢
- **選択テキストの初期検索ワード反映** — テキスト選択があって Cmd+F を押すと、その内容が検索入力に入って即検索実行

### 変更（破壊的ではない UX 変更）

- **Cmd+F の役割変更**: `outliner.js:2393` のキーハンドラを差し替え。以前は `.outliner-search-input`（ヘッダの `#tag, is:page, keyword` フィルタ）にフォーカスしていたが、新しいテキスト検索ボックスを開く動作に変更
- **Cmd+Shift+F を新設**: 旧 Cmd+F の動作（ヘッダフィルタへフォーカス）を Cmd+Shift+F に割り当て

### 非機能改善

- **サイドパネル MD エディタとの競合回避**: outliner の document-level Cmd+F/H ハンドラは、`.side-panel-editor-root` 内にフォーカスがある場合はスキップし、editor.js 側（MD エディタの `_handleSearchShortcut`）に委譲。`stopImmediatePropagation()` で同一 target 上の重複発火も防止
- **検索/置換ボックスの動的注入**: テンプレート（`outlinerWebviewContent.ts` / `notesWebviewContent.ts`）は変更せず、`outliner.js` の `setupTextSearchReplace()` で `document.body` に DOM を注入。両 webview で自動的に動作

### 主要な実装判断

- **レンダリング後テキストで検索し、source へ書き戻す方式を採用**  
  `node.text` は markdown マーカー付きの source（例: `**foo**`）として保存される。ユーザーが画面で見ている文字列（`foo`）で検索したいので、`stripInlineMarkers(node.text)` に対して regex を実行。マッチが見つかったら `renderedOffsetToSource()` で source 上のオフセットを求めて model.updateText する。これにより `**foo**` の `foo` を置換 → `**bar**` として書式維持が自然に実現する。ユーザー要件 4 に直接対応。
- **ハイライトは TreeWalker + splitText で複数テキストノードをまたぐマッチを分割**  
  MD エディタは `range.surroundContents()` を使い、複数要素にまたがるマッチは try/catch でスキップしている。outliner はインライン書式（bold/italic/link/tag）が常に描画されているため、マッチが複数ノードにまたがるケースが頻発する。`wrapRenderedRange()` で TreeWalker による text node 走査と `splitText` を組み合わせ、マッチ全体を正しく複数の `<span>` にラップする。
- **折畳は自動展開しない（ジャンプ時のみ展開）**  
  ユーザー要件 2 に従い、検索結果のカウントには折畳内マッチも含めるが、ツリーの見た目は変えない。Next/Prev でその match にジャンプする瞬間だけ `ensureNodeExpanded` で祖先チェーンを uncollapse し、`renderTree()` を再描画してスクロール。
- **検索/置換ボックスの動的注入でテンプレート改変を回避**  
  outliner は `outlinerWebviewContent.ts` と `notesWebviewContent.ts` の 2 箇所のテンプレートから呼ばれる。両方に HTML を書くのは二重管理になるため、`outliner.js` の init フローから `document.body` に要素を注入する方針。CSS は両 webview で共通ロードされる `outliner.css` に追加するだけで済む。
- **サイドパネル MD エディタとの競合回避**  
  outliner ページでは editor.js も side panel 用にロードされており、document-level の Cmd+F ハンドラを持つ。outliner のハンドラは `document.activeElement.closest('.side-panel-editor-root')` をチェックし、該当すれば outliner 側の処理をスキップ。逆に outliner 側で処理する場合は `stopImmediatePropagation()` で editor.js 側の発火を抑制。
- **Cmd+Shift+F への付替え**  
  `handleNodeKeydown` の既存 `switch(e.key)` は `!e.shiftKey` ガード内にあるため、`case 'f'` を修正しても shift キーでは到達しない。Cmd+Shift+F 用の分岐を Cmd+Shift+C の隣に新設（`outliner.js:2360` 付近）。

## DoD 検証結果

このリリースは `/sync` を経由しておらず、v11 artifact が未整備のため DoD 表は暫定。チャットで合意した 5 つの決定事項を検証基準として記載：

| # | 検証項目 | 結果 |
|---|---|---|
| 1 | subtext も検索/置換対象に含める | ✅ `performTextSearch` が node.subtext も走査 |
| 2 | 折畳ノード内マッチはカウントのみ、ジャンプ時のみ自動展開 | ✅ `ensureNodeExpanded` がジャンプ時のみ発動 |
| 3 | Replace All は 1 undo | ✅ `replaceAllTextMatches` で `saveSnapshot()` を冒頭 1 回のみ |
| 4 | インライン書式を維持した置換（`**foo**` → `**bar**`） | ✅ `renderedOffsetToSource` で source 位置に書き戻し |
| 5 | 検索スコープ = 現在のフォーカススコープ、なければ全ノード | ✅ `getScopedNodeIds()` が `currentScope` を参照 |

## 既知の制限事項・残課題

- **`.harness/` artifact 未同期**: 今回の機能は v11 requirement/design として正式化されていない。次回スプリント開始時に `/sync`（または `/requirement` → `/design`）で v11 として artifact 化することを推奨
- **自動テスト未追加**: Playwright による spec（`outliner-text-search-replace.spec.ts` 想定）が存在しない。次スプリントで追加する想定
- **編集中ノードのハイライトはスキップ**: `document.activeElement === el` の時は innerHTML を書き換えない設計（編集中の innerHTML 破壊を防ぐため）。blur 時に再描画される
- **現状、検索ボックスは `outliner.js` 内のグローバル変数（`textSearchMatches` 等）で状態管理**。複数 outliner を同時表示するシナリオは想定されていない（現 UI ではありえない）
