# 0.195.614

## 概要
Markdown 検索結果のジャンプが「常に1件目」に飛んでしまう問題を修正。N 番目のヒットを正しく指せるよう occurrence index を追加。

## 課題
0.195.613 で導入した `scrollToText` は TreeWalker で最初のヒットしか見ていなかった。同じファイル内で同一キーワードが複数ヒットする場合、ユーザーが3件目をクリックしても1件目に飛んでしまっていた。

## 原因
`fileResult.matches` 配列内で各マッチを区別する識別子を付けず、webview 側も第一ヒットで `break` していた。

## 対策

### 設計
クリックしたマッチの `fileResult.matches` 配列内インデックスを `occurrence` として全経路で引き回し、webview の TreeWalker は document order で N 件目に到達したところで停止する。

backend `searchMdFile` は raw markdown を行頭から順に走査して push するため `matches[]` の順序＝ document order。webview の TreeWalker (SHOW_TEXT) も document order なので原則対応する。

### 実装
1. [notes-file-panel.js:702-722](src/shared/notes-file-panel.js#L702-L722) で `forEach((match, matchIdx) => ...)` の `matchIdx` を `bridge.jumpToMdPage(..., query, matchIdx)` で渡す
2. [notes-host-bridge.js:187-196](src/shared/notes-host-bridge.js#L187-L196) `occurrence` フィールドを追加
3. [notes-message-handler.ts:25,659](src/shared/notes-message-handler.ts#L659) Platform 型と呼び出しを更新
4. [notesEditorProvider.ts:167-188](src/notesEditorProvider.ts#L167-L188) と [electron/src/main.ts:282-291](electron/src/main.ts#L282-L291) で `scrollToText` メッセージに `occurrence` を載せる
5. [editor.js:14069 付近](src/webview/editor.js#L14069) `scrollToText(query, occurrence)` を更新
   - 同一 textNode 内の複数ヒットも内側ループでカウント (`indexOf(needle, from)`)
   - N 番目に到達するまでスキップ
   - N 番目が見つからない場合は最初のヒットにフォールバック
