# Release 0.195.781 — Outliner undo/redo cursor fixes

## ハイライト

1. **cmd+z / cmd+shift+z で「実際に編集したノード」へカーソル復帰** — 子ノード追加・削除の取消や、Tab/Shift+Tab による indent/outdent の取消で、操作対象ノード自身に正しくフォーカスする（旧: 親ノードに飛んでしまう）
2. **cmd+z 1 回で重複 snapshot を一括 pop** — undo stack に積まれた連続同一 snapshot を 1 操作で消費。「最初の cmd+z で何も起きず、2 回目で undo 実行」現象を解消
3. **検索ボックスでの cmd+z 干渉を多層防御で完全抑止** — native input undo が outliner state を破壊する race を 5 層の防御で塞ぐ
4. **collapse 配下にあるノードへの undo フォーカスで自動展開** — undo 対象が折りたたみ中の親配下にある場合、全祖先を展開してから focus

## 詳細な変更内容

### 修正

#### `findDiffNodeIdInTarget()` の 4 phase 化（最重要）

undo/redo 時に「直前に何が変わったか」を snapshot 比較で特定するロジック。旧実装は children 配列 diff を最優先していたため、子ノード追加 / indent / outdent で `parent.children` が変化した時に常に親ノードが返り、フォーカスが親に飛んでしまっていた。

新実装は以下の優先順:

- **Phase 1 (最優先)**: text / subtext / collapsed / isPage / pageId / filePath / **parentId** の差分 → moved node 自身を返す
  - parentId 比較を含めることで indent/outdent が「moved node の差分」として直接検知される
- **Phase 2**: target だけに存在するノード（undo で復活 / redo で新規追加）
- **Phase 3**: target から消滅したノード（削除）→ 前兄弟 → 後続兄弟 → 親 の優先順で focus 候補
- **Phase 4 (最終手段)**: children 配列の差分（純粋な順序変更）

#### undo() / redo() の while-loop dedup

snapshot 連続同一を 1 操作で全て pop。旧: 1 回の cmd+z で 1 snapshot だけ pop → 重複があると「何も起きない」と感じる体験を解消。

#### 検索ボックスでの cmd+z 多層防御

Outliner の `searchInput` に focus 中の cmd+z で、native の input element undo history が `searchInput.value` を破壊する race を 5 層で防御:

1. document-level capture-phase keydown listener で `preventDefault`
2. `beforeinput` event の `inputType === 'historyUndo'` を `preventDefault`
3. `input` event で historyUndo を検知したら直前の値を復元
4. document-level setTimeout で再度復元
5. `focusin` redirect: cmd+z 後 300ms 以内の searchInput への focus を outliner 側に戻す

#### `markActivelyEditing()` を undo/redo に呼び出し

外部ファイル変更通知 (file watcher echo) が undo/redo 直後に発火して `undoStack` をリセットする race を防ぐ。1500ms idle で外部更新を queue。

#### `expandAncestorsAndFocus()` で全祖先展開

undo 対象ノードが collapse された親配下にある場合、`expandAncestorsAndFocus()` が全祖先を展開してから DOM focus する。
