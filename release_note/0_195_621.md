# 0.195.621

## 概要
0.195.620 で修正した Enter の挙動の追加調整。新ノードを必ず「すぐ下の行」に挿入し、現ノードが持っていた展開中の子は新ノード側へ付け替える。

## 課題
0.195.620 の修正後:
```
- a|
  - b
```
Enter →
```
- a
  - b
- |         ← 新ノードが子 b の下に置かれる
```
視覚的に「すぐ下の行」に現れないため直感に反する。

ユーザーが期待する挙動:
```
- a
- |
  - b       ← 新ノードがすぐ下、子 b は新ノードに付け替え
```

## 原因
[outliner.js:2481-2485](src/webview/outliner.js#L2481-L2485) の「通常 enter」経路は `model.addNode(node.parentId, node.id, afterText)` で新ノードを現ノードの兄弟として追加するが、現ノードが持つ子はそのまま。結果として新ノードは兄弟だが「子の下」に描画される。

## 対策

### 設計
Enter 時に現ノードが展開された子を持っていた場合、それらの子を新しい兄弟ノードへ付け替える。collapsed のときは付け替えない（折りたたまれた子は現ノードの所有物として保持すべき）。

これにより:
- 新ノードは視覚的に現行の直下に現れる
- 子ノードは階層関係を保ったまま新ノードに引き継がれる
- 折りたたまれたサブツリーは動かさない

### 実装
[outliner.js:2485-2500](src/webview/outliner.js#L2485-L2500):
```js
} else {
    model.updateText(node.id, beforeText);
    newNode = model.addNode(node.parentId, node.id, afterText);
    // 現ノードが展開された子を持っていたら、その子を新兄弟へ移す。
    if (node.children && node.children.length > 0 && !node.collapsed) {
        var movingChildren = node.children.slice();
        for (var mc = 0; mc < movingChildren.length; mc++) {
            var childNode = model.getNode(movingChildren[mc]);
            if (childNode) { childNode.parentId = newNode.id; }
        }
        newNode.children = movingChildren;
        node.children = [];
    }
}
```

## 挙動まとめ

| 操作 | 結果 |
|---|---|
| `a\|` + 子 b (展開) → Enter | `a` / `\|` (子b) |
| `a\|` + 子 b (折畳) → Enter | `a` (子b折畳) / `\|` |
| `\|a` + 子 b → Enter | `\|` / `a` (子b) |
| `a\|b` + 子 c → Enter | `a` / `b` (子c) |
