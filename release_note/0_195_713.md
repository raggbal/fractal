# Release v0.195.713

**リリース日:** 2026-04-19
**バージョン種別:** patch
**対応要件:** requirement v12 (拡張) / design v12 §12

## 概要

v0.195.712 で追加した Outliner D&D ファイル import を **VSCode Explorer からの D&D** にも対応させた拡張リリース。Finder 経路 (v0.195.712) と Explorer 経路の両方が並列で動作する。

## 変更内容

### 新機能

- **VSCode Explorer からの D&D** — VSCode の Explorer パネルからファイルをドラッグして outliner にドロップできるようになった。Explorer の D&D は `dataTransfer.files` が空で代わりに `application/vnd.code.uri-list` に `file://...` URI を入れるため、従来の Finder 経路では検出できなかった。
- **相対画像参照の解決**: Explorer 経路で `.md` を drop した場合、本文中の相対画像参照 (`![](sub/pic.png)` 等) が正しくコピーされパス書換される (Finder 経路の `skipRelativeImages` 制約なし)。
- **50MB 上限なし**: Explorer 経路はファイルバイトを webview を経由させずに host が直接読むため、サイズ制限がない。
- **非 local scheme の拒否**: `vscode-remote://` 等のリモート scheme URI は警告メッセージで拒否。

### 変更 (既存機能の維持)

- Finder 経路 (`'Files'` dataTransfer type) は従来通り動作。
- ⋮ メニュー版の「Import any files...」「Import .md files...」も従来通り。
- 既存の 25%/50%/25% ドロップ位置判定、フォルダ拒否、視覚フィードバックは両経路で共通。

### 破壊的変更

なし。

### 主要な実装判断

- **設計 v12 初版で却下した `application/vnd.code.uri-list` 利用を撤回**: min VSCode 1.85.0 で安定 API として確立されており、却下理由は弱かった。session-log.md に撤回理由を記録。
- **既存 `importFiles` / `importMdFiles` を直接呼び出す**: 新規 fs.writeFileSync 経路を作らず、メニュー版と完全に同じ内部ルートを共有 (DRY)。
- **webview 側 `dropFilesResult` ハンドラは変更ゼロ**: Explorer 経路の結果も同じメッセージ shape で返却、webview 側のノード作成ロジックを再利用。
- **非 file:// scheme の拒否**: `uri.scheme !== 'file'` で事前チェック、warning 通知。
